\section*{Introduction}
Ce projet a pour objectif d'étudier en profondeur la régression linéaire à travers plusieurs jeux de données présentant chacun une difficulté particulière. La démarche adoptée consiste à traiter chaque problème une première fois par nos propres moyens (niveau bac+3), puis à comparer cette approche avec celle que l'on obtient en s'appuyant sur une IA. Le but n'est pas d'opposer les deux méthodes mais d'identifier précisément ce que l'IA apporte en plus d'un raisonnement statistique d'étudiant.

\section{Régression linéaire simple}
%======================================================================

Cette section traite quatre jeux de données ($A$, $B$, $C$, $D$) couvrant chacun un cas typique : situation idéale, présence d'outliers, petit échantillon, et relation non-linéaire. Pour chaque dataset, on présente d'abord la démarche étudiant puis ce que l'IA apporte en complément.

%----------------------------------------------------------------------
\subsection{Cadre théorique}
%----------------------------------------------------------------------

Le modèle de régression linéaire simple s'écrit
\[
Y_i = a X_i + b + U_i, \qquad i = 1, \dots, n,
\]
où les $U_i$ sont supposés indépendants, de loi $\mathcal{N}(0, \sigma^2)$. Les estimateurs des moindres carrés ordinaires (MCO) sont
\[
\hat{a} = \frac{\sum_{i=1}^n (X_i - \bar{X})(Y_i - \bar{Y})}{\sum_{i=1}^n (X_i - \bar{X})^2}, \qquad \hat{b} = \bar{Y} - \hat{a}\bar{X}.
\]
Sous les hypothèses du modèle, $\hat{a}$ suit une loi normale et la statistique
\[
T = \frac{\hat{a}}{\widehat{\mathrm{SE}}(\hat{a})}
\]
suit une loi de Student à $n-2$ degrés de liberté sous $H_0 : a = 0$. La validation du modèle repose sur l'analyse des résidus studentisés, le QQ-plot de normalité, le test de Kolmogorov-Smirnov contre la loi $\mathcal{T}(n-3)$ et le graphique des résidus en fonction des valeurs ajustées.

%----------------------------------------------------------------------
\subsection{Dataset A : cas idéal}
%----------------------------------------------------------------------

\subsubsection{Approche étudiant}

On ajuste le modèle avec \texttt{lm()} et on lit la sortie de \texttt{summary()}. Le coefficient de détermination $R^2$ est très élevé, la pente $\hat{a}$ est très significativement non nulle (p-valeur $< 2 \cdot 10^{-16}$).

\begin{lstlisting}
modele_A <- lm(Y ~ X, data = A)
summary(modele_A)
cat("R² =", summary(modele_A)$r.squared, "\n")
\end{lstlisting}

Le tracé du nuage de points avec la droite de régression montre un alignement net (figure~\ref{fig:A_droite}).

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{figures/NuagePointA-droiteRegressionLineaire.png}
    \caption{Dataset A : nuage de points et droite ajustée.}
    \label{fig:A_droite}
\end{figure}

\paragraph{Diagnostic des résidus.} Le graphique des résidus studentisés en fonction des valeurs ajustées (figure~\ref{fig:A_residus}, gauche) ne présente aucun motif visible : les points se répartissent symétriquement autour de zéro, sans pattern en entonnoir, ce qui valide l'hypothèse d'homoscédasticité. Le QQ-plot (figure~\ref{fig:A_residus}, droite) suit la première bissectrice, indiquant la normalité des résidus.

\begin{lstlisting}
plot(fitted(modele_A), rstudent(modele_A),
     xlab = "ajustes", ylab = "residus stud.", main = "A - residus vs ajustes")
abline(h = 0, col = "red", lty = 2)

qqnorm(rstudent(modele_A), main = "A - QQ-plot")
qqline(rstudent(modele_A), col = "red")
\end{lstlisting}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.48\textwidth]{figures/residuStudent-A.png}
    \hfill
    \includegraphics[width=0.48\textwidth]{figures/QQ-plotA.png}
    \caption{Diagnostic des résidus du modèle ajusté sur le Dataset A.}
    \label{fig:A_residus}
\end{figure}

\paragraph{Test de Kolmogorov-Smirnov.} On teste la normalité des résidus studentisés contre la loi $\mathcal{T}(n-3)$ :

\begin{lstlisting}
n_A <- nrow(A)
ks.test(rstudent(modele_A), "pt", df = n_A - 3)
\end{lstlisting}

La p-valeur est largement supérieure à $0{,}05$ : on ne rejette pas l'hypothèse de normalité. Les quatre hypothèses du modèle sont validées.

\paragraph{Test de Student.} On calcule manuellement
\[
\hat{\sigma} = \sqrt{\tfrac{1}{n-2} \sum_i \hat{U}_i^2}, \quad \widehat{\mathrm{SE}}(\hat{a}) = \frac{\hat{\sigma}}{\sqrt{\sum_i (X_i - \bar{X})^2}},
\]
puis $t_{\mathrm{obs}} = \hat{a}/\widehat{\mathrm{SE}}(\hat{a})$. La valeur observée dépasse très largement le seuil critique $t_{0{,}975,\, n-2}$ : on rejette $H_0$, la pente est significative.

\subsubsection{Apport de l'IA}

\begin{itemize}
    \item \textbf{Vérification manuelle systématique.} L'IA propose de recalculer $\hat{a}$ et $\hat{b}$ directement à partir des formules du cours pour vérifier les sorties de \texttt{summary()}.

\begin{lstlisting}
x_bar <- mean(A$X); y_bar <- mean(A$Y)
a_chap <- sum((A$X - x_bar) * (A$Y - y_bar)) / sum((A$X - x_bar)^2)
b_chap <- y_bar - a_chap * x_bar
cat("calcul manuel -> a =", a_chap, "| b =", b_chap, "\n")
\end{lstlisting}

    \item \textbf{Tests formels supplémentaires.} En plus des outils graphiques, l'IA propose le test de Breusch-Pagan pour l'homoscédasticité et le test de Shapiro-Wilk pour la normalité, objectivant des conclusions qu'un étudiant tirerait uniquement du QQ-plot.

\begin{lstlisting}
library(lmtest)
bptest(modele_A)              # Breusch-Pagan : homoscedasticite
shapiro.test(residuals(modele_A))  # Shapiro-Wilk : normalite
\end{lstlisting}

    \item \textbf{Visualisation de la région de rejet.} Tracer la densité de Student avec la région de rejet remplie en rouge clarifie immédiatement la procédure du test (figure~\ref{fig:A_rejet}).

\begin{lstlisting}
sigma_chap <- sqrt(sum(residuals(modele_A)^2) / (n_A - 2))
se_a  <- sigma_chap / sqrt(sum((A$X - x_bar)^2))
t_obs <- a_chap / se_a
seuil <- qt(0.975, df = n_A - 2)

curve(dt(x, df = n_A - 2), from = -5, to = 5, n = 500,
      xlab = "t", ylab = "densite", main = "A - region de rejet")
xd <- seq(seuil, 5, length.out = 100)
xg <- seq(-5, -seuil, length.out = 100)
polygon(c(seuil, xd, 5),   c(0, dt(xd, n_A-2), 0), col = rgb(1,0,0,0.3), border = NA)
polygon(c(-5, xg, -seuil), c(0, dt(xg, n_A-2), 0), col = rgb(1,0,0,0.3), border = NA)
abline(v = c(-seuil, seuil), col = "red", lty = 2)
\end{lstlisting}

\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.65\textwidth]{figures/RegionRejetA.png}
    \caption{Région de rejet du test bilatéral sur la pente, Dataset A.}
    \label{fig:A_rejet}
\end{figure}

\begin{itemize}
    \item \textbf{Intervalle de prédiction.} L'IA propose \texttt{predict()} avec \texttt{interval = "prediction"} pour obtenir l'encadrement d'une nouvelle observation, bouclant proprement la chaîne modèle-inférence-prédiction.

\begin{lstlisting}
predict(modele_A, newdata = data.frame(X = c(0, 50)), interval = "prediction")
\end{lstlisting}

\end{itemize}

\subsection{Dataset B : présence d'outliers}
%----------------------------------------------------------------------

\subsubsection{Approche étudiant}

L'ajustement direct produit un $R^2$ correct mais le graphique des résidus studentisés (figure~\ref{fig:B_residus}, gauche) fait apparaître plusieurs points avec $|\text{résidu}| > 3$. Le QQ-plot dévie nettement aux extrémités (figure~\ref{fig:B_residus}, droite) et le test KS rejette la normalité.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.48\textwidth]{figures/residuStudent-B.png}
    \hfill
    \includegraphics[width=0.48\textwidth]{figures/QQplot-B.png}
    \caption{Diagnostic du modèle initial sur le Dataset B.}
    \label{fig:B_residus}
\end{figure}

\paragraph{Identification et retrait.} On identifie les outliers via le seuil $|\text{résidu studentisé}| > 3$, on les visualise sur le nuage (figure~\ref{fig:B_outliers}, gauche) puis on refit le modèle sur les données nettoyées (figure~\ref{fig:B_outliers}, droite).

\begin{lstlisting}
n_B <- nrow(B)
out <- which(abs(rstudent(modele_B)) > 3)
cat("outliers detectes :", out, "\n")

B2 <- B[-out, ]
modele_B2 <- lm(Y ~ X, data = B2)

cat("pente avant :", coef(modele_B)[2],
    "| apres :", coef(modele_B2)[2], "\n")
\end{lstlisting}

Le QQ-plot et le test KS du modèle nettoyé sont alors satisfaisants. La différence de pente illustre la sensibilité de la MCO aux valeurs extrêmes.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.48\textwidth]{figures/nuagePointOutliersRouge-B.png}
    \hfill
    \includegraphics[width=0.48\textwidth]{figures/NuagePointB-nettoye.png}
    \caption{Avant/après retrait des outliers, Dataset B.}
    \label{fig:B_outliers}
\end{figure}

\subsubsection{Apport de l'IA}

\begin{itemize}
    \item \textbf{Distance de Cook.} L'IA introduit la distance de Cook pour distinguer les points \emph{aberrants} (résidu élevé) des points \emph{influents} (gros impact sur les coefficients). Un point peut être atypique sans être influent, et inversement.

\begin{lstlisting}
cook_B <- cooks.distance(modele_B)
plot(cook_B, type = "h", main = "B - distance de Cook")
abline(h = 4 / n_B, col = "red", lty = 2)
\end{lstlisting}

    \item \textbf{Régression robuste.} Plutôt que de supprimer des observations, \texttt{rlm()} (package \texttt{MASS}) pondère automatiquement les points selon leur résidu (M-estimateurs), évitant la décision arbitraire de retrait.

\begin{lstlisting}
library(MASS)
modele_B_rob <- rlm(Y ~ X, data = B)
cat("MCO :", coef(modele_B)[2],
    "| rlm :", coef(modele_B_rob)[2],
    "| sans outliers :", coef(modele_B2)[2], "\n")
\end{lstlisting}

    \item \textbf{Comparaison visuelle des trois approches} (figure~\ref{fig:B_IA}).

\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.48\textwidth]{figures/DistanceCook-B.png}
    \hfill
    \includegraphics[width=0.48\textwidth]{figures/MCO-rlm-noOutliers-B.png}
    \caption{Distance de Cook et comparaison MCO / rlm / sans outliers, Dataset B.}
    \label{fig:B_IA}
\end{figure}

\subsection{Dataset C : petit échantillon ($n = 50$)}
%----------------------------------------------------------------------

\subsubsection{Approche étudiant}

Le modèle s'ajuste sans difficulté apparente et les hypothèses semblent satisfaites graphiquement (figure~\ref{fig:C_main}). Toutefois, la lecture du \texttt{summary()} met en évidence une erreur standard $\widehat{\mathrm{SE}}(\hat{a})$ nettement plus large que celle du Dataset A. La largeur de l'intervalle de confiance à 95\% sur la pente est environ $\sqrt{n_A / n_C} \approx 4{,}5$ fois plus grande, comme attendu d'après la décroissance en $1/\sqrt{n}$.

\begin{lstlisting}
n_C <- nrow(C)
cat("SE(a) -> A :", summary(modele_A)$coefficients[2, 2],
               "| C :", summary(modele_C)$coefficients[2, 2], "\n")
cat("largeur IC95 -> A :", diff(confint(modele_A)[2, ]),
                  "| C :", diff(confint(modele_C)[2, ]), "\n")
\end{lstlisting}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.48\textwidth]{figures/NuagePointC-DroiteRegression.png}
    \hfill
    \includegraphics[width=0.48\textwidth]{figures/ResiduStudent-C.png}
    \caption{Ajustement et résidus pour le Dataset C ($n = 50$).}
    \label{fig:C_main}
\end{figure}

\subsubsection{Apport de l'IA}

\begin{itemize}
    \item \textbf{Tests formels sur petit échantillon.} En petit $n$, les diagnostics graphiques sont peu fiables. L'IA insiste sur le test de Breusch-Pagan pour confirmer l'homoscédasticité au-delà du graphique.

    \item \textbf{Erreurs standards robustes (sandwich).} En cas de doute sur l'homoscédasticité, \texttt{vcovHC()} fournit une matrice de variance robuste, raffinement standard en économétrie qu'un étudiant ne connaît pas en général.

\begin{lstlisting}
library(lmtest); library(sandwich)
bptest(modele_C)
coeftest(modele_C, vcov = vcovHC(modele_C, type = "HC1"))
\end{lstlisting}

    \item \textbf{Bootstrap pour l'intervalle de confiance.} Avec $n = 50$, l'IA propose un bootstrap par rééchantillonnage ($R = 2000$) pour un intervalle de confiance qui ne repose pas sur l'hypothèse de normalité asymptotique.

\begin{lstlisting}
library(boot)
boot_fn <- function(d, i) coef(lm(Y ~ X, data = d[i, ]))
set.seed(123)
b_C <- boot(C, boot_fn, R = 2000)
cat("IC classique :", confint(modele_C)[2, ], "\n")
cat("IC bootstrap :", boot.ci(b_C, type = "perc", index = 2)$percent[4:5], "\n")
\end{lstlisting}

\end{itemize}

\subsection{Dataset D : relation non-linéaire en V}
%----------------------------------------------------------------------

\subsubsection{Approche étudiant -- tentative directe}

L'ajustement d'un modèle linéaire simple donne un $R^2$ très faible et le graphique des résidus présente un motif en V flagrant (figure~\ref{fig:D_simple}). Le test KS rejette la normalité. Le modèle linéaire est manifestement inadapté.

\begin{lstlisting}
modele_D <- lm(Y ~ X, data = D)
cat("R² =", summary(modele_D)$r.squared, "\n")
ks.test(rstudent(modele_D), "pt", df = nrow(D) - 3)
\end{lstlisting}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.48\textwidth]{figures/NuagePointD-DroiteRegression.png}
    \hfill
    \includegraphics[width=0.48\textwidth]{figures/ResiduStudent-D.png}
    \caption{Tentative d'ajustement linéaire direct sur le Dataset D.}
    \label{fig:D_simple}
\end{figure}

\subsubsection{Approche étudiant -- découpage en deux régressions}

L'observation du nuage de points suggère un point de rupture autour de $X = 30$. On scinde le jeu de données en deux sous-ensembles et on ajuste deux régressions séparées. Les diagnostics deviennent satisfaisants sur chaque sous-modèle et les $R^2$ remontent à des valeurs très acceptables (figure~\ref{fig:D_split}).

\begin{lstlisting}
seuil_X <- 30
D_g <- D[D$X <  seuil_X, ]
D_d <- D[D$X >= seuil_X, ]
mod_g <- lm(Y ~ X, data = D_g)
mod_d <- lm(Y ~ X, data = D_d)
cat("R² gauche :", summary(mod_g)$r.squared,
    "| R² droite :", summary(mod_d)$r.squared, "\n")
\end{lstlisting}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{figures/NuagePoint2droite-D.png}
    \caption{Ajustement par morceaux du Dataset D, point de rupture fixé à $X = 30$.}
    \label{fig:D_split}
\end{figure}

\subsubsection{Apport de l'IA}

\begin{itemize}
    \item \textbf{Estimation automatique du point de rupture.} Le choix $X = 30$ est visuel et arbitraire. La régression segmentée (package \texttt{segmented}) estime le point de rupture $\hat{\psi}$ avec son erreur standard, transformant un choix \emph{ad hoc} en une véritable estimation statistique.

\begin{lstlisting}
library(segmented)
modele_D_seg <- segmented(modele_D, seg.Z = ~X, psi = 30)
cat("rupture estimee :", modele_D_seg$psi[, "Est."], "\n")
cat("R² segmentee :", summary(modele_D_seg)$r.squared, "\n")
\end{lstlisting}

    \item \textbf{Modélisation par variable indicatrice.} L'IA propose aussi un seul \texttt{lm()} avec $Z = \max(X - 30, 0)$, cas particulier de régression multiple qui unifie le traitement.

\begin{lstlisting}
D$Z <- pmax(D$X - 30, 0)
mod_pcw <- lm(Y ~ X + Z, data = D)
summary(mod_pcw)
\end{lstlisting}

    \item \textbf{Comparaison de $R^2$} entre les trois modèles : simple ($R^2 \approx 0$), par morceaux, et segmenté (figure~\ref{fig:D_seg}).

\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{figures/RegressionSegmente-D.png}
    \caption{Régression segmentée sur le Dataset D, point de rupture estimé automatiquement.}
    \label{fig:D_seg}
\end{figure}

\subsection{Synthèse}
%----------------------------------------------------------------------

Les quatre datasets illustrent quatre situations pratiques rencontrées en régression simple :
\begin{itemize}
    \item Dataset A : tout fonctionne, on valide la chaîne complète (ajustement, test, résidus, prédiction).
    \item Dataset B : la MCO est sensible aux outliers. L'approche étudiant consiste à les retirer ; l'IA propose des méthodes plus robustes (rlm, distance de Cook).
    \item Dataset C : la précision des estimateurs se dégrade en $1/\sqrt{n}$. L'IA propose des outils adaptés au petit $n$ (bootstrap, erreurs standards robustes).
    \item Dataset D : le modèle linéaire est inadapté pour une relation en V. Le découpage manuel fonctionne mais l'IA introduit la régression segmentée qui formalise et automatise la démarche.
\end{itemize}
